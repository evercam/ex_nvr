## Overview

An **LPR Event** (License Plate Recognition event) is a structured record of a vehicle license plate detected by a camera. Unlike generic [events](event.md), which carry an arbitrary `metadata` map, LPR events have a dedicated schema with typed fields for the plate number, capture time, vehicle direction, list type (whitelist/blacklist), and rich metadata including bounding box coordinates, confidence score, and vehicle attributes.

LPR events enter ExNVR through two paths:

1. **Camera polling** — The `ExNVR.Devices.LPREventPuller` GenServer runs as part of the per-device supervision tree (when `enable_lpr` is true on the [device](device.md) settings). It polls the camera's HTTP API every 10 seconds, fetching new plate detections since the last known capture time. This path supports **Hikvision** and **Milesight** cameras, each with vendor-specific HTTP client implementations that know how to parse the camera's proprietary response formats.

2. **Webhook ingestion** — External systems can push LPR events via `POST /api/devices/:device_id/events/lpr`, authenticated with a webhook token. Currently only Milesight camera webhook payloads are parsed (via `ExNVRWeb.LPR.Parser.Milesight`); other vendors return a 404.

Each LPR event can have an associated **plate image** — a JPEG crop of the detected license plate. These images are stored on the filesystem under the device's `thumbnails/lpr/` directory, named `{event_id}_{plate_number}.jpg`. The images are fetched from the camera alongside the event data (via the vendor HTTP client) or decoded from the webhook payload.

The UI provides a dedicated LPR events page (`/events/lpr`) with a filterable, paginated table showing plate images, device name, capture time, plate number, and direction. Users can preview the full camera snapshot at the capture time and play a 10-second video clip centered on the event.

## Data model

### `ExNVR.Events.LPR` (`ui/lib/ex_nvr/events/lpr.ex`)

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `id` | integer (auto) | — | Primary key |
| `capture_time` | `:utc_datetime_usec` | — | When the plate was detected, required |
| `plate_number` | `:string` | — | The detected plate text, required |
| `direction` | `Ecto.Enum` | `:unknown` | `:in` (approaching), `:away` (departing), or `:unknown` |
| `list_type` | `Ecto.Enum` | — | `:white`, `:black`, or `:other` |
| `plate_image` | `:string` | — | **Virtual field** — not persisted; populated on-demand with base64-encoded plate image |
| `device_id` | `:binary_id` | — | FK to [device](device.md) |
| `inserted_at` | `:utc_datetime_usec` | — | Row creation time |
| `updated_at` | `:utc_datetime_usec` | — | Row update time |

**Flop integration**: Filterable by `capture_time`, `plate_number`, `direction`, `list_type`, and `device_id`. Sortable by `capture_time` and `plate_number`. Supports `device_name` join field. Page-based pagination with default limit 20, max 50. Default sort: descending by `capture_time`.

### Embedded schema: `LPR.Metadata`

| Field | Type | Notes |
|-------|------|-------|
| `confidence` | `:float` | Detection confidence score |
| `bounding_box` | `{:array, :float}` | Normalized `[x1, y1, x2, y2]` coordinates (values in 0..1 range) |
| `plate_color` | `:string` | Color of the plate |
| `vehicle_type` | `:string` | Type of vehicle (e.g. car, truck) |
| `vehicle_color` | `:string` | Color of the vehicle |

The bounding box is validated to have exactly 4 values, all within the range `[0, 1]`. The coordinates are normalized against the image resolution — the Milesight parser divides pixel coordinates by resolution width/height, and the Hikvision client passes raw XML data that the camera normalizes.

### Plate image storage

Plate images are JPEG files stored on the device's filesystem:

```
{storage_address}/ex_nvr/{device_id}/thumbnails/lpr/{event_id}_{plate_number}.jpg
```

The filename is generated by `LPR.plate_name/1`. Images are written during `Events.create_lpr_event/3` — the directory is created on demand via `File.mkdir/1`.

## API surface

### REST API

| Method | Path | Auth | Action |
|--------|------|------|--------|
| `POST` | `/api/devices/:device_id/events/lpr` | Webhook token | Create LPR event from webhook |
| `GET` | `/api/events/lpr` | User auth | List/filter LPR events (paginated) |

**Creating LPR events via webhook** (`EventController.create_lpr/2`):
- The controller checks `Device.vendor/1` and dispatches to a vendor-specific parser
- Currently only `:milesight` is supported; all other vendors return `{:error, :not_found}`
- `ExNVRWeb.LPR.Parser.Milesight.parse/2` extracts: `plate_number` from `"plate"`, `capture_time` from `"time"` (naive datetime interpreted in device timezone), `direction` (mapped from `"approach"`/`"away"`), `list_type` (from `"type"`), metadata including `bounding_box` (pixel coordinates normalized by resolution), `confidence`, `vehicle_color`, `vehicle_type`, and `plate_image` (base64-decoded from `"plate_image"`)

**Listing LPR events** (`EventController.lpr/2`):
- Returns `{meta, data}` with pagination metadata
- Accepts optional `include_plate_image=true` query parameter to include base64-encoded plate images in the response (each image is read from disk via `Events.lpr_event_thumbnail/1`)

### LiveView page

| Path | Module | Purpose |
|------|--------|---------|
| `/events/lpr` | `LPREventsListLive` | Dedicated LPR events browser |

The LPR events page features:
- Filterable table with device dropdown, capture time range, and plate number (LIKE matching)
- Plate image thumbnails in each row (falls back to a default "no plate image" placeholder)
- Capture times displayed in the device's timezone
- **Snapshot preview**: clicking the eye icon loads a precise snapshot from the recording at the event's capture time via `/api/devices/:id/snapshot?time=...&method=precise`
- **Clip preview**: clicking the video icon plays a 10-second HLS clip centered on the event (5 seconds before to 5 seconds after capture time) via `/api/devices/:id/hls/index.m3u8`

## Business logic

### `ExNVR.Events` context — LPR functions

**`create_lpr_event/3`**:
1. Builds the changeset with `device_id` injected
2. Inserts with `on_conflict: :nothing` — if a duplicate is detected (likely same `capture_time` + `plate_number`), the insert is silently skipped
3. Only proceeds to save the plate image if the insertion returned a non-nil `id` (i.e., the row was actually inserted, not a conflict no-op)
4. Writes the plate image JPEG to the device's `thumbnails/lpr/` directory

**`last_lpr_event_timestamp/1`**: Returns the most recent `capture_time` for a device, used by the `LPREventPuller` to know where to resume polling.

**`lpr_event_thumbnail/1`**: Reads the plate image file from disk and base64-encodes it. Returns `nil` if the file doesn't exist.

**`list_lpr_events/2`**: Flop-powered paginated listing. Accepts an `include_plate_image: true` option that maps over results and attaches base64 plate images to the virtual `plate_image` field.

### Camera polling — `ExNVR.Devices.LPREventPuller`

A GenServer started as part of the per-device supervision tree when `device.settings.enable_lpr` is `true` and the device has an HTTP URL configured.

**Polling cycle**:
1. On init, queries `Events.last_lpr_event_timestamp/1` to find the resume point
2. Every 10 seconds (`@pulling_interval`), calls `Devices.fetch_lpr_event/2`
3. `Devices.fetch_lpr_event/2` routes to the vendor-specific HTTP client (`Hik` or `Milesight`) with the device URL, credentials, timezone, and last event timestamp
4. The vendor client fetches events from the camera's HTTP API, filtering by timestamp
5. Plate images are fetched concurrently (up to 4 at a time via `Task.async_stream`)
6. Events and their plate images are saved via `Events.create_lpr_event/3`
7. The `last_event_timestamp` state is updated to the max `capture_time` of newly stored events

### Vendor HTTP clients

**Hikvision** (`ExNVR.Devices.Cameras.HttpClient.Hik`):
- Endpoint: `POST /ISAPI/Traffic/channels/1/vehicledetect/plates` with XML body containing the `AfterTime` filter
- Parses XML response via `SweetXml` — extracts `captureTime`, `plateNumber`, `picName`, `direction`
- Direction mapping: `"forward"` → `:in`, `"reverse"` → `:away`
- Plate images fetched from `/doc/ui/images/plate/{picName}.jpg`
- Capture time format: `YYYYMMDDTHHMMSS+offset`, parsed in device timezone

**Milesight** (`ExNVR.Devices.Cameras.HttpClient.Milesight`):
- Endpoint: `GET /cgi-bin/operator/operator.cgi?action=get.lpr.lastdata&format=inf`
- Parses INF-format response (key=value pairs, 10 lines per entry)
- Direction mapping: `"1"` → `:in`, `"2"` → `:away`
- Plate images fetched from `/LPR/{path}`
- Filters events client-side using `DateTime.after?/2` against the last event timestamp

### Webhook parsing

The `ExNVRWeb.LPR.Parser` behaviour defines the contract: `parse(term(), timezone()) :: {map(), plate_image()}`.

Currently only `ExNVRWeb.LPR.Parser.Milesight` implements this for webhook ingestion. The Milesight webhook parser extracts structured fields from JSON, normalizes bounding box coordinates from pixels to `[0..1]` range, parses confidence as a float, and base64-decodes the plate image from the `"plate_image"` field.

## Related concepts

- [device](device.md) — LPR events are associated with a device; the device's settings control whether LPR polling is enabled
- [event](event.md) — Generic events share the same context module (`ExNVR.Events`) and controller, but use a different schema and database table

## Business rules

- **Duplicate detection via `on_conflict: :nothing`** — If an LPR event with the same unique constraint already exists, the insert is silently skipped and no plate image is saved. This prevents the polling mechanism from creating duplicate records.
- **Plate image is not in the database** — Plate images are stored on the filesystem, not as a database column. The `plate_image` field on the schema is virtual and populated on-demand via `lpr_event_thumbnail/1`.
- **Webhook LPR creation is vendor-gated** — `EventController.create_lpr/2` only supports Milesight cameras. Devices with other vendors (or no vendor) return `{:error, :not_found}`.
- **Polling requires `enable_lpr` and an HTTP URL** — The `LPREventPuller` is only started when `device.settings.enable_lpr` is `true` and the device has a configured HTTP `url`. Without these, no polling occurs.
- **Capture time is always stored in UTC** — Both polling clients and webhook parsers convert from the device's local timezone to UTC before storage. The UI converts back to the device's timezone for display.
- **Bounding box normalization** — All bounding box coordinates are normalized to `[0, 1]` range regardless of source. The Milesight webhook parser divides pixel coordinates by resolution dimensions; the changeset validates that all values fall within this range.
